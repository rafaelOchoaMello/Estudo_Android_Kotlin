58.Argumentos Nomeados

- Uma característica muito útil do kotlin são os 'argumentos nomeados', consistindo
basicamente de um display da assinatura dos parâmetros definidos pelo método.

- Isto não impacta diretamente na qualidade ou otimização do código, mas facilita muito seus 
entendimento, uma vez que é comum vermos métodos recebendo parâmetros, e não sabemos ou lembramos
com certeza o que cada um deles significa no contexto do mesmo.

Exemplo: Considerando o método a seguir
	
	fun mostraChefeFuncionario(nomeChefe:String, nomeFuncionario:String): String {
		return "O ${nomeFuncionario} é funcionario de ${nomeChefe}"
	}

	//Em outras linguagens, teríamos uma chamada assim

	mostraChefeFuncionario("João", "Maria")

	//Por mais que possa parecer simples, imagine que tu nunca viu a implementação do 
	método antes. Nesse caso, sem abrir o local onde o método está para fazer uma 
	análize, como saberíamos quem é funcionário e quem é o chefe ?


	//Em Kotlin, na sua IDE específica (IntelliJ), há a feature dos argumentos nomeados,
	fazendo assim que o display da chamada fique

	mostraChefeFuncionario(nomeChefe:"João", nomeFuncionario:"Maria")

- É importante notar que esta é uma característica da IDE do kotlin, NÃO da linguagem 
de programação em sí.
