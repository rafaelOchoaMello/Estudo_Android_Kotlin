81&82.Gets'n sets

- Assim como temos na maioria das linguagens, há uma implementação para gets e sets 
em kotlin, apesar da sintaxe ser um pouquinho diferente.

Exemplo::

data class Job(
	//Tanto faz o construtor...
) {

    //Isso significa que podemos ver o valor, mas não é possível modificar diretamente
    // o set não é exatamente um modificador, apenas uma restrição, um método normal já basta
    // para realizar o controle necessário para modificação
    var jobBonus: Int = 0
        private set

    init {}

    fun setJobBonus(newBonus: Int) {
        if (newBonus > jobBonus)
            jobBonus = newBonus
        else
            println("Valor do bonus deve ser maior que o atual!")
    }

    fun showJobData(): String = "Classe: ${name} | Nível requerido: ${levelRequire} | Bonus: ${jobBonus}"
    fun showJobSpells() {
        for (spell in spellList)
            spell.showSpellData()
    }
}


- Caso seja desejado que não seja possível ver o valor do jobBonus, poderíamos simplesmente 
deixar o atributo privato, ou então deixar o get igual a como está o set.
